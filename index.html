Abstract

This project focuses on predicting ICD (diagnosis codes) and CPT (procedure codes) from clinical records. Existing methods do not fully use temporal information or data from multiple documents. Most models use only a single document, such as a discharge summary, and do not capture changes in a patient's condition during admission or important information from different clinical documents (Mullenbach et al., 2018; Huang et al., 2022). Our goal is to solve this problem.

As our initial approach, we organize the data at the admission level and design two baseline models for comparison. Baseline 1 is a BioClinicalBERT-based (Alsentzer et al., 2019) multi-label classification model that uses only a discharge summary as input. Baseline 2 combines multiple clinical documents, such as discharge and radiology reports, to use more information. To handle long text more effectively, we use the Longformer model (Beltagy et al., 2020).

In early experiments, Baseline 1 used 86,146 admissions and achieved Micro-F1 of 0.4615, Macro-F1 of 0.4660, Precision@5 of 0.2340, and Precision@8 of 0.1603. Baseline 2 used a smaller sample of 2,000 admissions and achieved Precision@5 of 0.1034 and Precision@8 of 0.0767.

We also design a BioClinicalBERT + LSTM (Hochreiter and Schmidhuber, 1997) model to include temporal reasoning by processing clinical documents in time order. This model used 101,655 admissions and achieved a test Micro-F1 of 0.5335, Macro-F1 of 0.4857, Precision@5 of 0.2357, and Precision@8 of 0.1624 after 3 epochs of training, outperforming Baseline 1 across all metrics.

Introduction / Background / Motivation

The goal of this project is to automatically predict diagnosis codes (ICD) and procedure codes (CPT) from clinical records to improve the efficiency of healthcare systems.

Most existing automated coding systems use only a single document, such as a discharge summary, as input (Mullenbach et al., 2018; Huang et al., 2022; Yang et al., 2022). However, in real clinical settings, a patient's condition changes over time, and important information is spread across multiple documents, such as radiology reports and other clinical notes.

Because of this, single-document approaches have clear limitations. Therefore, this study aims to address these limitations to reduce the workload of medical staff and improve hospital efficiency.

Approach

We designed two baseline models and one temporal model for initial performance comparison. Baseline 1 is a single-document model that uses only a discharge summary as input. Baseline 2 is a multi-document model that combines multiple clinical documents. Our temporal model processes multiple documents in time order to reflect changes in the patient's condition during admission.

We used the following data. The discharge.csv file contains discharge summary notes and is used as the main clinical text. The key columns are hadm_id, charttime, and text. The radiology.csv file contains radiology reports, and its key columns are hadm_id, charttime, and text.

To connect ICD labels, we used diagnosis.csv and edstays.csv. The stay_id in diagnosis.csv is linked to hadm_id through edstays.csv, which allows us to match admissions with ICD codes.

Baseline 1

In Baseline 1, we built a model that predicts ICD codes from only one discharge summary. We matched notes and labels using hadm_id and collected 86,146 admissions. The model is based on BioClinicalBERT (Alsentzer et al., 2019), and we added a dropout layer and a linear classification head to predict the presence of the top 50 ICD codes. The data were split into Train, Validation, and Test sets as 60,302 / 12,922 / 12,922. This baseline is simple and stable, but it cannot use the many kinds of clinical information created during admission.

Baseline 2

In Baseline 2, we designed a multi-document model to use more information. For each admission, discharge and radiology documents were grouped by hadm_id and sorted by charttime. The documents were then merged into a single long text input. We used a Longformer-based (Beltagy et al., 2020) multi-label classification model to predict 50 ICD codes. This approach uses more information than a single-document model and reflects the structure of clinical data. However, in early experiments, it used a small dataset and simple document merging, so it could not capture temporal relationships.

Temporal Model

In our temporal model, we explicitly include temporal reasoning by processing multiple documents in time order within the same hadm_id. Radiology notes and discharge summaries are sorted by charttime, and up to five notes are used as one sequence for each admission. Each document is first encoded with BioClinicalBERT (Alsentzer et al., 2019), and then a two-layer unidirectional LSTM (Hochreiter and Schmidhuber, 1997) models the patient's state changes across time. Using this structure, we collected 101,655 admissions and split them into Train / Validation / Test as 71,158 / 15,248 / 15,249.

Challenges

There were several implementation challenges. In Baseline 1, there was an ID mismatch problem when connecting discharge.csv and diagnosis.csv. Because discharge notes and ICD tables did not directly share the same ID format, we used edstays.csv as a bridge to convert stay_id into hadm_id. In Baseline 2 and the temporal model, it was also important to keep data consistency while merging multiple files. In addition, input length became much larger when documents were combined or processed as sequences, which increased memory usage.

To handle this, in the temporal model we limited the maximum number of notes per admission to five (MAX_NOTES = 5) and used a batch size of 8 so that training could fit within GPU memory.

Results

We evaluated model performance in a multi-label classification setting using Micro F1, Macro F1, and Precision@K (Zhang and Zhou, 2014). In multi-label medical coding, a single patient may have multiple ICD codes at the same time, making it important to evaluate not only overall prediction accuracy but also the precision of top-ranked predictions.

Baseline 1 used only discharge summaries and achieved a Micro F1 of 0.4615. Baseline 2 used both discharge and radiology data but achieved lower performance. This reflects the small sample size (1,171 admissions after preprocessing and label filtering), as well as the simple document concatenation approach, which does not model temporal relationships between documents.

In contrast, our temporal model outperformed Baseline 1 across all metrics, achieving a test Micro F1 of 0.5335 (+7.2 percentage points). Training loss decreased steadily across epochs (0.1103 → 0.0733 → 0.0654), and validation Micro F1 improved consistently (0.4636 → 0.5089 → 0.5373), confirming that temporal reasoning over multiple clinical notes provides meaningful signal beyond single-document classification.

Both models showed more stable predictions on frequent ICD codes, while prediction errors were more common on rare or complex diagnoses. The Macro F1 of our temporal model (0.4857) indicates that performance on rare ICD codes remains limited, which we plan to address in future work.

Baseline 2 could not capture relationships between documents because it simply merged multiple notes into one text. In contrast, the temporal model processed notes in chronological order using an LSTM (Hochreiter and Schmidhuber, 1997), which allowed it to model diagnostic progression across the admission period.

The number of admissions used in each model differs due to input structure and computation constraints. Baseline 2 required longer input sequences for the Longformer (Beltagy et al., 2020), and GPU memory constraints required training on a smaller sampled dataset. Because of this, these results should be seen as preliminary findings that show the strengths and limits of each model structure, rather than a perfectly fair comparison. In future work, we plan to compare the models under the same data setting.

Conclusion and Future Work

In this study, we addressed the problem of predicting ICD codes from clinical records and compared a single-document baseline, a multi-document baseline, and a temporal model. Baseline 1 was designed as the simplest model using only a discharge summary, while Baseline 2 tested whether using more documents could improve performance. Our temporal model was designed to process documents in time order with an LSTM (Hochreiter and Schmidhuber, 1997) so that it can better reflect changes in the patient's condition over time.

We built the models step by step in order to clearly test the effect of temporal modeling. Baseline 1 helped us measure how much can be predicted from a single document. Baseline 2 helped us check whether using more information alone could improve performance. The results show that simple document merging is not sufficient, and that explicitly modeling time order provides a meaningful advantage.

Our temporal model demonstrated clear improvement over the single-document baseline, achieving a test Micro F1 of 0.5335 compared to 0.4615 for Baseline 1 (+7.2 percentage points). This confirms that modeling diagnostic progression across multiple clinical notes captures complementary information that a single discharge summary cannot provide. The lower Macro F1 indicates that performance on rare ICD codes remains limited, which we plan to address in future work.

At the current stage, we focus on ICD prediction to confirm the stability of the modeling pipeline. We plan to extend the final model to CPT code prediction as well. Approaches that rely only on discharge summaries do not fully reflect patient state changes across time (Mullenbach et al., 2018), so future models should make better use of clinical data created at different time points.

Future work will focus on the following directions:

  1. Add structured data such as lab results, vital signs, and medications as additional inputs
  2. Introduce preprocessing methods to detect copy-forward problems and remove repeated information
  3. Extend the system from ICD prediction to joint ICD and CPT prediction
  4. Fully train Baseline 2 (Longformer) on a larger dataset and perform a fairer comparison across all models

Mentor Feedback & Progress

1. Summary of Mentor Feedback

The mentor emphasized that comparing only Baseline 1 and the temporal model is not sufficient. Since using multiple documents naturally improves performance, the key comparison should be between Baseline 2 (multi-document without temporal ordering) and the temporal model. This helps clearly demonstrate whether incorporating temporal information truly improves performance.

The mentor also suggested increasing the dataset size for Baseline 2 to improve the reliability and confidence of the results. In addition, due to the length of clinical documents and token limitations, we currently use a limited number of notes. The mentor recommended exploring possible approaches such as input compression or keyword filtering, while also noting that it is acceptable to clearly state these limitations in the project.

Overall, the mentor commented that the project is making good progress and is on track for the final presentation.

2. Plan to Address Feedback

Based on the mentor’s feedback, we plan to strengthen our experiments on Baseline 2 by increasing the dataset size and improving the reliability of the results. We will focus on a clear comparison between Baseline 2 and the temporal model to validate our main claim that temporal modeling provides additional value.

We will also clearly describe system limitations, such as token constraints and the limited number of notes, in our final report. If time allows, we will explore simple techniques such as input compression or filtering methods to incorporate more document information into the model.

Analysis

For ICD-only prediction, Baseline 1 (BioClinicalBERT) achieved a Micro-F1 score of 0.4342 and Precision@5 of 0.4229, while Baseline 2 (Longformer) performed worse with a Micro-F1 of 0.4051 and Precision@5 of 0.3977. In contrast, the temporal model (BioClinicalBERT + LSTM) significantly improves performance, achieving a Micro-F1 of 0.5018 and Precision@5 of 0.4712.

For CPT-only prediction, Baseline 1 achieved a Micro-F1 of 0.5373 and Baseline 2 achieved 0.5612. The temporal model shows competitive performance with a Micro-F1 of 0.5018 but achieves the highest Precision@5 (0.4924), indicating better ranking quality of predicted codes.

In the joint ICD+CPT setting, the temporal model achieves 0.5018 Micro-F1 for ICD and 0.6024 for CPT. This demonstrates that a unified temporal framework can effectively handle both diagnosis and procedure prediction within a single model.

Overall, these results highlight that temporal multi-note modeling consistently improves ICD prediction, while CPT prediction remains more stable due to structured procedural patterns. Additionally, modeling multiple notes over time improves top-ranked prediction accuracy, emphasizing the importance of capturing clinical progression.
