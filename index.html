Abstract

This project focuses on predicting ICD (diagnosis codes) and CPT (procedure codes) from clinical records. Existing methods do not fully use time information or data from multiple documents. Our goal is to solve this problem.

Most previous models use only a single document, such as a discharge summary. Because of this, they cannot capture important information, such as changes in the patient’s condition during admission or information from radiology reports, lab results, and other documents.

To solve this problem, we organize the data at the admission level and design two baseline models for comparison. Baseline 1 is a BioClinicalBERT-based multi-label classification model that uses only a discharge summary as input. Baseline 2 combines multiple clinical documents, including discharge and radiology notes, to use more information. In Baseline 2, we use a Longformer model to handle long text more effectively.

In early experiments, Baseline 1 used 86,146 admissions and achieved Micro-F1 of 0.4615, Macro-F1 of 0.4660, Precision@5 of 0.2340, and Precision@8 of 0.1603. Baseline 2 used a smaller sample of 2,000 admissions and achieved Precision@5 of 0.1034 and Precision@8 of 0.0767. This shows the limits of simple document merging and limited data.

We also design a BioClinicalBERT + LSTM model to include temporal reasoning by processing multiple clinical documents in time order. This model used 101,655 admissions and achieved a validation Micro-F1 of 0.4636 after 1 epoch.

Introduction / Background / Motivation

The goal of this project is to automatically predict diagnosis and procedure codes from clinical records. Medical coding is important for hospital operations and insurance billing, but many parts of this process are still done by hand, which takes time and cost.

Most existing auto-coding systems mainly use only one document, such as a discharge summary. However, in real clinical settings, a patient’s condition changes over time, and important information is spread across multiple documents, such as radiology reports and other notes.

Because of this, a single-document approach has an important limit. If this project works well, it may help reduce the workload of medical staff and improve hospital efficiency.

Approach

Before building our final temporal modeling system, we designed two baseline models and one temporal model for performance comparison. Baseline 1 is a single-document model that uses only one discharge summary as input. Baseline 2 is a multi-document model that combines several clinical documents to use richer information. Our temporal model processes multiple documents in time order so that it can better reflect changes in the patient’s condition during admission.

We used the following data. The discharge.csv file contains discharge summary notes, which are the main text for final clinical summaries. The main columns are hadm_id, charttime, and text. The radiology.csv file contains radiology reports with time-related imaging information. Its main columns are also hadm_id, charttime, and text. To connect ICD labels, we used diagnosis.csv together with edstays.csv. In diagnosis.csv, the main columns are stay_id and icd_code. In edstays.csv, stay_id is linked to hadm_id, which allows us to connect admissions with ICD labels.

Baseline 1

In Baseline 1, we built a model that predicts ICD codes from only one discharge summary. We matched notes and labels using hadm_id and collected 86,146 admissions. The model is based on BioClinicalBERT, and we added a dropout layer and a linear classification head to predict the presence of the top 50 ICD codes. The data were split into Train, Validation, and Test sets as 60,302 / 12,922 / 12,922. This baseline is simple and stable, but it cannot use the many kinds of clinical information created during admission.

Baseline 2

In Baseline 2, we designed a multi-document model to use richer information. For each admission, discharge and radiology documents were grouped by hadm_id and then sorted by charttime. After that, multiple documents were merged into one long text input. We used a Longformer-based multi-label classification model to predict 50 ICD codes. This approach can use more information than a single-document model and better reflects the structure of real clinical data. However, in our early experiment, it used only a small sample and simple document merging, so it could not fully capture temporal relationships.

Temporal Model

In our temporal model, we explicitly include temporal reasoning by processing multiple documents in time order within the same hadm_id. Radiology notes and discharge summaries are sorted by charttime, and up to five notes are used as one sequence for each admission. Each document is first encoded with BioClinicalBERT, and then a two-layer unidirectional LSTM models the patient’s state changes across time. Using this structure, we collected 101,655 admissions and split them into Train / Validation / Test as 71,158 / 15,248 / 15,249.

Challenges

There were several implementation challenges. In Baseline 1, there was an ID mismatch problem when connecting discharge.csv and diagnosis.csv. Because discharge notes and ICD tables did not directly share the same ID format, we used edstays.csv as a bridge to convert stay_id into hadm_id. In Baseline 2 and the temporal model, it was also important to keep data consistency while merging multiple files. In addition, input length became much larger when documents were combined or processed as sequences, which increased memory usage.

To handle this, in the temporal model we limited the maximum number of notes per admission to five (MAX_NOTES = 5) and used a batch size of 8 so that training could fit within GPU memory.

Results

We evaluated model performance in a multi-label classification setting using Micro F1, Macro F1, and Precision@K. In medical coding, one patient may have multiple ICD codes at the same time, so it is important not only to predict all codes perfectly, but also to check how accurate the top predicted codes are.

Although Baseline 2 used more information, its performance was lower than Baseline 1. This reflects the small sample size, the simple document merging method, and the fact that temporal relationships were not modeled. Important information may also have been diluted during the long-text processing step. In contrast, the temporal model showed a slightly better Micro F1 than Baseline 1 on validation after 1 epoch, which suggests that temporal information may be helpful. However, its Macro F1 was lower, likely because rare ICD codes were not learned well enough yet. Since the temporal model was trained for only one epoch, more training may further improve performance.

Qualitatively, both models were more stable on frequent ICD codes, while prediction errors appeared more often on rare or complex diagnoses.

Baseline 2 could not fully capture relationships between documents because it simply merged multiple notes into one text. In contrast, the temporal model included radiology notes and used time order, which allowed it to use more information from the admission process. This suggests that additional training may lead to further improvements.

We used Precision@K because this is a useful metric in multi-label medical coding. A single patient may have several ICD codes at the same time, and in practice it is often more important that the top few predicted codes are correct than that every possible code is predicted perfectly.

The number of admissions used in each model is different. This difference comes from the input structure and computation limits of each model. In particular, Baseline 2 used a Longformer-based structure to process long text, so GPU memory limits required us to train on a small sampled dataset. After preprocessing and label filtering, the final number of admissions for Baseline 2 became 1,171.

Because of this, these results should be seen as preliminary findings that show the strengths and limits of each model structure, rather than as a perfectly fair one-to-one comparison under the same data setting.

In future work, we plan to compare the models under a more controlled and equal data setting so that performance differences can be analyzed more clearly.

Conclusion and Future Work

In this study, we addressed the problem of predicting ICD codes from clinical records and compared a single-document baseline, a multi-document baseline, and a temporal model. Baseline 1 was designed as the simplest model using only a discharge summary, while Baseline 2 was designed to test whether using more documents could improve performance. Our temporal model was designed to process documents in time order with an LSTM so that it can better reflect changes in the patient’s condition over time.

We built the baselines step by step in order to clearly test the effect of temporal modeling. Baseline 1 helped us measure how much can be predicted from only one document. Baseline 2 helped us check whether performance could improve by using more information. The results suggest that simple information merging is not enough, and that modeling time order is important.

In early experiments, Baseline 1 showed stable performance, while Baseline 2 was limited by simple document merging and a small amount of data. Our temporal model showed similar or slightly better Micro F1 than the baseline, which suggests that time-aware modeling has promise. The lower Macro F1 suggests that rare ICD codes still need better learning.

At the current stage, we focus on ICD prediction to confirm the stability of the modeling pipeline. We plan to extend the final model to CPT code prediction as well. In particular, approaches that rely only on discharge summaries cannot fully reflect patient state changes across time, so future models should make better use of clinical data created at different time points.

Future work will focus on the following directions:

  1. Train the temporal model longer (at least 3 epochs) and report final test performance
  2. Add structured data such as lab results, vital signs, and medications as additional inputs
  3. Introduce preprocessing methods to detect copy-forward problems and remove repeated information
  4. Extend the system from ICD prediction to joint ICD and CPT prediction
  5. Fully train Baseline 2 (Longformer) on a larger dataset and perform a fairer comparison
